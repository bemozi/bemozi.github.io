## Contributing to Statim

We appreciate your interest in contributing to [Statim](https://bemozi.github.io). Please review these guidelines thoroughly before submitting to ensure a smooth collaboration.

## Reporting Bugs

To submit a [comprehensive bug report](https://github.com/bemozi/bemozi.github.io/issues/new/choose), please include:

*  A **clear and descriptive title**.
*  The **steps to reproduce** the bug (e.g., "Go to X, click Y, get error Z").
*  The **expected behavior** versus the **actual behavior**.
*  Your **operating system** and **browser version**.
*  Any relevant **screenshots** or console output.

## Suggesting Enhancements

We highly value suggestions for new features and improvements. To [propose an enhancement](https://github.com/bemozi/bemozi.github.io/pulls):

*  Open an issue with the label `[Feature Request]`.
*  Clearly describe the feature, explaining **why** it would be valuable to Statim users and how it aligns with the local-first and auditable principle.
